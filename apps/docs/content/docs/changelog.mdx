Motebit ships 12 packages to npm — 11 Apache-2.0 packages on the [public surface](/docs/concepts/public-surface) and 1 BSL-1.1 reference runtime. Each carries its own `CHANGELOG.md` written by [Changesets](https://github.com/changesets/changesets), so "what changed" lives next to the code that changed.

## Authoritative sources

| Where | What you find there |
|---|---|
| [GitHub Releases](https://github.com/motebit/motebit/releases) | Human-readable release notes, one per published package per version. The right place to browse "what shipped" as a reader. |
| Per-package `CHANGELOG.md` | Machine-maintained, byte-stable record of every release for that package. Linked per-row below. |
| [`.changeset/`](https://github.com/motebit/motebit/tree/main/.changeset) directory | In-flight changes that haven't shipped yet — every PR lands a changeset describing the bump. |
| Root [`CHANGELOG.md`](https://github.com/motebit/motebit/blob/main/CHANGELOG.md) | Pre-`1.0.0` history (entries `0.1.0` through `0.3.0`) preserved from before Changesets adoption. Pointer document for everything after. |

The doctrine is **one canonical source per synchronization invariant**. A root CHANGELOG that duplicated the per-package ones drifted between `0.3.0` and `1.0.0` — so the root file now points at the per-package files instead of competing with them. Drift defense `check-changeset-discipline` (#43) enforces that every PR ships a substantive changeset.

## Workspace-private packages

The 51 workspace-private packages (everything not in the table above) are pinned to `0.0.0-private` per the [public-surface doctrine](/docs/concepts/public-surface). They have no semver claim and no independent changelog — their behavior changes ship as part of `motebit`'s release cadence, documented in the `motebit` package's `CHANGELOG.md`. The Changesets `ignore` list locks them at the sentinel so internal-dependency bumps can't creep them upward.

## Versioning policy

Each published package follows [semver 2.0.0](https://semver.org/). Breaking changes require a `major` changeset with a non-empty `## Migration` section per `check-changeset-discipline`. The Apache-2.0 packages use `check-api-surface` to enforce that the type-extracted baseline (`packages/<name>/etc/<name>.api.md`) cannot drift without an explicit major bump. The `motebit` CLI uses `check-cli-surface` against `apps/cli/etc/cli-surface.json` for the operator-facing surface (subcommands, flags, exit codes, paths).

The 12 packages all currently sit at `1.0.0` — the coordinated initial release.
