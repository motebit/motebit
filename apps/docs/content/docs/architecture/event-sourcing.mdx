title: Event Sourcing
description: Append-only log, version clocks, tombstoning, and compaction.

Motebit uses event sourcing as its persistence model. Every state change, memory formation, identity event, and tool execution is recorded as an immutable event in an append-only log.

## Events

Every event is an `EventLogEntry`:

```ts
interface EventLogEntry {
  event_id: string;
  motebit_id: string;
  device_id: string;
  timestamp: number;
  event_type: EventType;
  payload: Record<string, unknown>;
  version_clock: number;
  tombstoned: boolean;
}
```

### Event types

19 event types covering the full lifecycle:

| Category | Types |
|---|---|
| Identity | `identity_created`, `device_registered` |
| State | `state_updated` |
| Memory | `memory_formed`, `memory_updated`, `memory_deleted` |
| Conversation | `message_sent`, `message_received` |
| Tools | `tool_called`, `tool_result` |
| System | `session_started`, `session_ended`, `error_occurred` |
| Sync | `sync_started`, `sync_completed`, `conflict_detected` |
| Compaction | `snapshot_created`, `compaction_performed` |

## Version clocks

Each motebit has a monotonically increasing version clock. Every event gets the next clock value:

```ts
const clock = await identityManager.incrementClock(motebitId);
// clock: 1, 2, 3, ...

const event = {
  event_id: generateId(),
  motebit_id: motebitId,
  device_id: deviceId,
  timestamp: Date.now(),
  event_type: EventType.StateUpdated,
  payload: { attention: 0.8 },
  version_clock: clock,
  tombstoned: false,
};

await eventStore.append(event);
```

Version clocks enable:
- **Ordering** — Events are always processed in clock order, regardless of which device produced them
- **Cursoring** — Sync queries "events after clock N" for efficient delta sync
- **Conflict detection** — Two events from different devices with overlapping clocks indicate concurrent modification

## Querying

Filter events by type, time range, or clock position:

```ts
const events = await eventStore.query({
  motebit_id: "my-motebit",
  event_types: [EventType.MemoryFormed, EventType.MemoryDeleted],
  after_version_clock: 100,
  limit: 50,
});
```

## Replay

Reconstruct state from the event log:

```ts
await eventStore.replay(motebitId, (event) => {
  switch (event.event_type) {
    case EventType.StateUpdated:
      applyStateUpdate(event.payload);
      break;
    case EventType.MemoryFormed:
      addMemory(event.payload);
      break;
    // ...
  }
});
```

Events are replayed in version clock order.

## Tombstoning

Events are never physically deleted — they're tombstoned:

```ts
await eventStore.tombstone(eventId, motebitId);
```

A tombstoned event has `tombstoned: true`. Queries skip it by default. This preserves the audit trail while removing the event from active processing.

## Compaction

The log grows indefinitely. Compaction prunes old events:

```ts
const removedCount = await eventStore.compact(motebitId, beforeClock);
```

The flow:
1. Create a snapshot of current state
2. Record a `snapshot_created` event
3. Delete all events with `version_clock < snapshot.clock`
4. Record a `compaction_performed` event

After compaction, state is reconstructed from the snapshot + events after the snapshot clock.

Auto-compaction triggers when event count exceeds the configured `compactionThreshold` (default: 1000).

## Storage adapters

| Adapter | Use case |
|---|---|
| `InMemoryEventStore` | Tests |
| SQLite adapter (via `@motebit/persistence`) | Production |
| `HttpEventStoreAdapter` | Sync relay communication |
| `WebSocketEventStoreAdapter` | Real-time sync |
| `EncryptedEventStoreAdapter` | Encrypted sync |

All adapters implement the same `EventStoreAdapter` interface:

```ts
interface EventStoreAdapter {
  append(entry: EventLogEntry): Promise<void>;
  query(filter: EventFilter): Promise<EventLogEntry[]>;
  getLatestClock(motebitId: string): Promise<number>;
  tombstone(eventId: string, motebitId: string): Promise<void>;
  compact?(motebitId: string, beforeClock: number): Promise<number>;
  countEvents?(motebitId: string): Promise<number>;
}
```
