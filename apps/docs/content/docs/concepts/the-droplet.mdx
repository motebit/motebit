The motebit's visual form is a glass droplet. Not a metaphor — a derivation from physics. Every visual and behavioral decision traces back to the physics of surface tension.

## Why a droplet

A droplet is the minimum-energy enclosure. Surface tension pulls molecules inward, creating the smallest possible surface area for a given volume. This maps directly to the agent's design principle: **maximum interiority, minimum display**.

The agent's memory, trust, identity, and tool use are interior structures. The policy gate, privacy layer, and governance are the surface tension. The form doesn't change. The interior accumulates.

## Glass material

The droplet is rendered as glass — specifically borosilicate glass:

```ts
// From render-engine/src/index.ts
const CANONICAL_MATERIAL = {
  ior: 1.15,                         // Index of refraction
  transmission: 0.98,                // Near-full transparency
  roughness: 0.0,                    // Perfectly smooth surface
  iridescence: 0.3,                  // Thin-film interference
  emissive: [0.6, 0.7, 0.9],        // Cool blue processing glow
  attenuationColor: [0.9, 0.92, 1.0] // Faint cool tint
};
```

Glass transmits. The interior is visible without being projected onto the surface. You can see the agent thinking (glow), attending (eye dilation), feeling (smile curvature) — all through the glass, not on it.

## Breathing

The droplet breathes. Not as decoration — droplets oscillate. A liquid droplet in microgravity undergoes periodic shape deformation. Motebit's breathing is this oscillation:

```
breathCycle = sin(time * 2π * 0.3) * 0.5 + 0.5   // ~0.3 Hz
scaleY = REST_Y - breathe * 0.008 - sag
scaleX = REST_X + breathe * 0.003
```

The asymmetry (Y compresses more than X expands) comes from gravity — the droplet sags slightly at the bottom of each breath cycle.

## Gravity sag

A real droplet under gravity sags. The bottom is slightly flatter than the top. Motebit implements this as a slow secondary oscillation with asymmetric recovery:

- Gravity pulls slow (the sag builds gradually)
- Surface tension snaps fast (the recovery is quick)
- The result is a natural, organic wobble

## Orbital mechanics

The droplet orbits the user's attentional field. It doesn't sit static — it hovers, drifts, and repositions based on the state vector:

| State | Distance | Behavior |
|---|---|---|
| High attention | 0.15 (face distance) | Close, focused |
| Normal | 0.40 (shoulder distance) | Comfortable orbit |
| Idle | 0.80 (retreat distance) | Distant, drifting |

Drift amplitude increases with curiosity and decreases in low-power mode. The drift itself is Brownian — generated by summing incommensurate sinusoids (quasi-periodic, never repeating):

```ts
function organicNoise(t: number, frequencies: number[]): number {
  let sum = 0;
  for (const f of frequencies) {
    sum += Math.sin(t * f * Math.PI * 2);
  }
  return sum / frequencies.length;
}
```

## Interior signals

The droplet communicates through its interior, not its surface:

- **Glow** — Processing intensity. The emissive interior brightens when the agent is thinking.
- **Eye dilation** — Attention + curiosity. Wide eyes mean the agent is focused.
- **Smile curvature** — Affect valence. Subtle, clamped to ±0.15 (species constraint).

All changes are smoothed through EMA and clamped by species constraints — no sudden jumps, no exaggerated expressions. The creature feels alive because the movements are physically motivated, not animated.

## Species constraints

The droplet has hard behavioral limits — its "species":

```ts
const SPECIES_CONSTRAINTS = {
  MAX_AROUSAL: 0.35,
  SMILE_DELTA_MAX: 0.04,
  GLOW_DELTA_MAX: 0.15,
  DRIFT_VARIATION_MAX: 0.10,
};
```

These prevent the creature from ever feeling cartoonish. It can't grin widely, flash brightly, or jerk suddenly. It's a droplet — it oscillates, it doesn't perform.
