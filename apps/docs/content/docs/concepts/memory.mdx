title: Memory
description: Semantic graph with cosine similarity retrieval and half-life decay.
---

Memory is what separates a sovereign agent from a stateless chatbot. Motebit's memory system is a semantic graph — nodes are memories, edges are relationships, and retrieval uses embedding similarity with confidence decay over time.

## Memory nodes

Each memory is a `MemoryNode`:

```ts
interface MemoryNode {
  node_id: string;
  content: string;
  embedding: number[];       // vector for similarity search
  confidence: number;        // 0-1, decays over time
  sensitivity: SensitivityLevel;
  half_life: number;         // milliseconds (default: 7 days)
  created_at: number;
  last_accessed: number;
  tombstoned: boolean;
}
```

Memories are formed from LLM output via XML tags. The AI emits `<memory>` tags in its response, which the runtime parses and stores:

```xml
<memory confidence="0.9" sensitivity="personal">
  User prefers dark mode and uses vim keybindings
</memory>
```

## Memory formation

```ts
const memory = await memoryGraph.formMemory(
  { content: "user prefers dark mode", confidence: 0.9, sensitivity: SensitivityLevel.Personal },
  embedding,                 // float[] from embedding model
  7 * 24 * 60 * 60 * 1000   // 7-day half-life
);
```

Formation emits an audit event so every memory has a traceable origin.

## Retrieval

Retrieval is a two-pass process:

1. **Filter** — Exclude tombstoned memories and those below a confidence threshold
2. **Rerank** — Score by similarity, decayed confidence, and recency

```ts
const results = await memoryGraph.retrieve(queryEmbedding, {
  limit: 10,
  minConfidence: 0.1,
});
```

The scoring formula:

```
score = similarity * 0.5 + decayedConfidence * 0.3 + recencyBoost * 0.2
```

Where:
- `similarity` is cosine similarity between the query embedding and the memory embedding
- `decayedConfidence` applies half-life decay: `initial * 0.5^(elapsed / halfLife)`
- `recencyBoost` favors recently accessed memories: `1 / (1 + age / 24h)`

## Half-life decay

Memories don't persist at full strength forever. Confidence decays with a half-life (default 7 days):

```ts
function computeDecayedConfidence(
  initial: number,
  halfLife: number,
  elapsedMs: number
): number {
  return initial * Math.pow(0.5, elapsedMs / halfLife);
}
```

A memory stored at confidence 0.9 with a 7-day half-life:
- After 7 days: 0.45
- After 14 days: 0.225
- After 28 days: 0.056

Memories that fall below the retrieval threshold effectively fade, but they're never deleted — they can be reinforced if accessed again.

## Graph edges

Memories can be linked with typed relationships:

```ts
await memoryGraph.link(sourceId, targetId, RelationType.CausedBy, 0.8);
```

Relation types: `Related`, `CausedBy`, `FollowedBy`, `ConflictsWith`, `Reinforces`, `PartOf`.

Edges let the system understand not just individual facts but the structure between them — causal chains, contradictions, hierarchies.

## Sensitivity levels

Every memory carries a sensitivity level that controls retention and display:

| Level | Retention | Display |
|---|---|---|
| `None` | Infinite | Allowed |
| `Personal` | 365 days | Allowed |
| `Medical` | 90 days | Blocked |
| `Financial` | 90 days | Blocked |
| `Secret` | 30 days | Blocked |

The [privacy layer](/docs/architecture/privacy-and-policy) enforces these rules. Sensitive memories can only be accessed through governed channels, and deletion produces a cryptographic certificate.
