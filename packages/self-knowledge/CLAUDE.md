# @motebit/self-knowledge

Interior tier of the answer engine. A motebit's pre-built lexical index of
its own public docs — `README.md`, `DROPLET.md`, `THE_SOVEREIGN_INTERIOR.md`,
`THE_METABOLIC_PRINCIPLE.md`. When a user asks "what is Motebit?" or "what are
you?" the motebit looks here first, before `web_search`.

Layer 1. BSL-1.1. **Zero runtime deps.** Pure BM25 over a committed corpus —
no embedding model, no network, no tokens, no CDN. The corpus lives in
`src/corpus-data.ts` (generated by `scripts/build-self-knowledge.ts`). Query
path is a single file: chunk BM25 scores → top-k.

## Rules

1. **The corpus is generated, never hand-edited.** `src/corpus-data.ts`
   carries a banner and a checksum. Regenerate via
   `pnpm build-self-knowledge` whenever the source docs change. The CI drift
   gate would live here if/when the motebit's self-description becomes
   spec-load-bearing.
2. **No embeddings, no model downloads.** At ~15 heading-sized chunks,
   motebit-specific terms ("motebit", "droplet", "sovereign interior",
   "metabolic") are unique enough that BM25 matches the user's query
   directly. Swapping to vectors is a follow-up only if recall evidence
   demands it; do not pre-optimize.
3. **Corpus is self-attested, not receipt-attested.** An interior citation
   in a `CitedAnswer` carries `source: "interior"` and no `receipt_task_id`
   — the committed corpus + its hash are the provenance. Verifiers
   cross-check by re-hashing the corpus module at audit time.
4. **Static at build time.** Every motebit binary (CLI, desktop, web,
   spatial) ships the same corpus. A relay running an older motebit with
   an older corpus is a legitimate state — this package is not dynamic.

## What NOT to add

- Embeddings or vector search. That's `@motebit/memory-graph`'s job for
  user-derived memory. Self-knowledge is different — small, static, keyword-
  dense.
- Loaders for `docs/*` subdirectory files. The corpus covers the four
  top-level self-description docs only. Deeper docs belong to Ring-3 web
  research, not interior recall.
- Any I/O. The package imports `CORPUS_DATA` from a generated TS module
  and does pure in-memory ranking.
- Runtime fetches of GitHub README (that's Ring 3, and it would defeat the
  offline-always guarantee).

## Consumers

- `@motebit/tools` — wraps `querySelfKnowledge` in the `recall_self` builtin
  tool, registered alongside `web_search` and `recall_memories` in every
  Ring-1 surface (web, CLI, spatial).
- `services/research` — checks `recall_self` first before falling to the
  web-search atom. Returns a `CitedAnswer` with `source: "interior"` citations
  when the query clears the confidence threshold.
