# @motebit/protocol

The open protocol surface. Apache-2.0 (permissive floor), Layer 0, **zero monorepo dependencies**.

## Rules

1. **No I/O, no algorithms that bind the runtime.** Export only types, enums, branded ID casts, constants, and deterministic math (semiring algebra, canonical JSON, hash primitives). If an export decides, adapts, routes, prioritizes, governs, or stores anything, it belongs in BSL ‚Äî not here.
2. **`check-deps` enforces permissive-floor purity.** Imports from BSL packages are a CI failure. New non-trivial function exports must be added to `PERMISSIVE_ALLOWED_FUNCTIONS` in `scripts/check-deps.ts` with review justification.
3. **Spec Wire format types live here.** Every type named under a `#### Wire format (foundation law)` subsection in any `spec/*.md` must be exported from this package. `check-spec-coverage` enforces the mapping.
4. **Cryptosuite registry is law.** `SuiteId` in `src/crypto-suite.ts` is a closed string-literal union. Adding a suite is additive (new entry + new dispatch arm in `@motebit/crypto`). Renaming or removing one is a wire-format break.
5. **Interfaces at this layer are the interoperable contract.** `SettlementRail`, `GuestRail`, `SovereignRail`, `ChainAnchorSubmitter`, `CredentialSource`, `ServerVerifier` ‚Äî a third party building an alternative implementation binds to these, not to BSL. Closed literal-union registries promoted to **registered registries** (`SuiteId`, `TokenAudience`, `ContentArtifactType`, `TaskShape`, `SensitivityLevel`, `EventType`, `SettlementMode` at land ‚Äî see `REGISTERED_REGISTRIES` in `scripts/check-closed-registry-canonical.ts`) carry the canonical **eight-artifact set** per [`docs/doctrine/registry-pattern-canonical.md`](../../docs/doctrine/registry-pattern-canonical.md): closed-type + frozen `ALL_*` iteration array + `is*` type guard + test file + per-registry coverage gate + gate registration + perturbation probe + drift-defenses inventory entry. The meta-gate `check-closed-registry-canonical` watches the family as a whole ‚Äî adding a registry without one of the eight artifacts fails CI. Closely-related closed unions that are _not_ registered registries (`ToolMode`, `ComputerActionKind`, sub-axes of registered registries) follow the structural-lock pattern with bespoke coverage shapes (`check-tool-modes`, `check-computer-dispatcher-modes`, etc.) ‚Äî the registry pattern is for interop-law typed vocabularies that meet four criteria (interop law, multi-consumer, wire-format presence, real or anticipated drift). Adding a registry is intentional protocol-level work: union entry + named constant + iteration entry + gate update + doctrine + `REGISTERED_REGISTRIES` append.
6. **Constant naming follows the law-vs-convention distinction.** Interop-law constants (`TRUST_LEVEL_SCORES`, semiring identities) ship unprefixed; reference-implementation defaults that implementers MAY override (`REFERENCE_TRUST_THRESHOLDS`) ship with a `REFERENCE_` prefix. Rationale + rename mechanics: [`docs/doctrine/protocol-model.md`](../../docs/doctrine/protocol-model.md) ¬ß "Naming: interop law vs reference default".

Details on the three-layer model (permissive floor / BSL / accumulated state), the operational test, and protocol-shaped events: [`docs/doctrine/protocol-model.md`](../../docs/doctrine/protocol-model.md).
