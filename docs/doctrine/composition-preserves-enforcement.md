# Composition preserves enforcement

> **Motebit must prove not only that its components are correct, but that its intended guarantees remain enforced when those components are composed into the deployed system.**

This is the obligation the gate suite does not individually carry. Every gate proves a _local_ property of a part; none of them proves that the part is actually reached, with the right inputs, in the artifact that ships. The gap between those two is where a month of real defects lived ‚Äî so the obligation gets named, and its enforcement mechanism gets built, rather than being rediscovered one production incident at a time.

The two load-bearing words are deliberate. **"Remain enforced"** ‚Äî not "survive": a guarantee can still _exist_ in the source and never be _acted on_ by the running system. **"Deployed system"** ‚Äî not "running under test": a composition that CI assembles correctly can still be assembled wrong in the shipped image (different config resolution, unwired keys, a dropped call). The sentence fails exactly the defects below, which is what makes it a test and not a slogan.

## The flaw class, with a birthdate

Trace the commit history for the family whose own messages admit a capability existed but did not fire ‚Äî `inert / dormant / shadowed / silent / lost`. It does not scatter. It clusters, and the first cluster has a date:

- **Mid-June 2026** ‚Äî the CLI-entry sev-1s: [`motebit-verify`](../../packages/verify) inert for every installed invocation (symlink'd-bin entry guard), the relay key silently dead via its bin, cloud-usage debits silently dropped. A correct binary, an unreached entry point.
- **Late July 2026** ‚Äî the routing-transcript / federation arc: the transcript producer dormant in deployed molecules because delegator signing keys were unwired (`#357`); the minted transcript lost at egress in branch churn (`#358`); the `#188` discover-signature sunset **inert in production because `server.ts` shadowed the constant** (`#359`); and the settlement-authority payer seam, whose first version enforced only at the relay ‚Äî _after_ the payer had already broadcast ‚Äî and so protected records but not funds ([`settlement-authority-binding`](settlement-authority-binding.md)).

The flaw began mid-June because that is when the system crossed a complexity threshold and the dominant bug **changed shape**: from _logic-wrong_ (a part computes the wrong thing; a unit test catches it) to _enforcement-not-local_ (the part is correct and green, but the deployed path does not reach it). The root cause is structural, not a discipline lapse: in a 53-package DAG a feature's **definition** ‚Äî type, crypto law, conformance vector, all passing ‚Äî and its **activation** ‚Äî the deployed process invoking it with resolved config and wired keys ‚Äî live in _different packages_. Every gate we had proved definition. Nothing proved activation.

## Honest calibration ‚Äî what the gates do and don't prove

Do not overstate the starting position. Our gates predominantly prove **local** properties ‚Äî dependency-layer membership ([`check-deps`](../../scripts/check-deps.ts)), generated-file consistency, closed-registry canonicality, declaration presence, formatting, wire-format coverage. A few reach further ([`check-gates-effective`](../../scripts/check-gates-effective.ts) perturbs each gate to prove it still fires; [`check-money-authority`](../../scripts/check-money-authority.ts) enforces the R4 standing-authority composition). But **none individually proves that the complete deployed path preserves every intended guarantee.** That framing matters: it makes the gap _structural_ ‚Äî a missing category of proof ‚Äî not a coverage hole to be closed by raising a threshold.

Symmetrically, do not overstate the fix. A harness earns the claim "proves enforcement in the deployed system" only if it builds the **actual production artifact**, boots it through the real entry point, resolves production-equivalent configuration, uses real credential and key wiring, crosses the persistence and process boundaries, exercises the external interfaces, verifies forbidden side effects do not occur ‚Äî **and goes red when each critical connection is deliberately severed.** A harness that composes the parts in-process is just another _modeled_ composition, which is precisely the thing that passed for `#357`. The severing test is the whole point: an activation check that does not fail when you re-shadow the constant proves nothing.

## The response is two-part, and detection is the weaker half

The first response already shipped: the effective-config harness ([`#360`](../../services/relay/src/relay-config.ts), `buildRelayConfigFromEnv` + `SECURITY_BOUNDARY_DEFAULTS` + `probeSecurityBoundaries`) ‚Äî it proves the _resolved_ security-boundary config at the enforcement point, not the exported constant, and [`check-security-default-wiring`](../../scripts/check-security-default-wiring.ts) forbids a hard-coded literal shadowing a canonical default. That is real, and it is the **detection** half.

But detection is the weaker half, and naming it as the whole answer would be a mistake. _A test detects a shadowed constant; a better configuration architecture makes shadowing structurally impossible._ This is [`runtime-invariants-over-prompt-rules`](runtime-invariants-over-prompt-rules.md) ‚Äî make illegal states unrepresentable ‚Äî applied to composition: one resolution path, no second literal that _can_ diverge, so the shadow does not compile rather than getting caught after the fact. So the doctrine is a pair, ranked:

1. **Reduce the seams where enforcement can disappear.** Prefer a structure where the guarantee cannot be composed away ‚Äî a single resolution path, a producer that cannot be constructed without its key, an egress that is the return type rather than a side assignment. Structural impossibility beats detection wherever it is affordable.
2. **Prove composition for the seams that remain.** Where a seam is irreducible, an **activation-conformance** pass exercises the deployed artifact end to end and proves each pipeline link ‚Äî identity ‚Üí authorization ‚Üí policy ‚Üí action ‚Üí receipt ‚Üí settlement ‚Üí trust ‚Äî is _live_, with the deliberate-severing test as its integrity check.

## The pattern, seeded ‚Äî needle-severing vs behavioral-severing

Inc 1 establishes the mechanism on one real incident, and the distinction it draws is the whole point. Every one of `#357`/`#358`/`#359` already had a _static_ defense: a needle in a gate ([`check-routing-transcript-emission`](../../scripts/check-routing-transcript-emission.ts)'s eight chain links; [`check-security-default-wiring`](../../scripts/check-security-default-wiring.ts)'s canonical-constant reference), and [`check-gates-effective`](../../scripts/check-gates-effective.ts) already perturbs those gates to prove they fire. Call that **needle-severing**: reintroduce the failure as an absent _source string_ and prove a gate catches it. It is necessary but not sufficient ‚Äî and `#358` is the proof, in its own commit message: _"the result-type field and every other link survived, so the emission gate stayed green"_ while the runtime behavior was broken. A string can be present and the deployed path still not execute it.

So the skeleton is **behavioral-severing**: reintroduce the failure as the real runtime _condition_, run the actual path, and prove the guarantee's assertion goes red. Two increments build it for `#357`, and the distinction between them is itself doctrine:

- **Inc 1 ‚Äî the runtime-level negative fixture** ([`execute-granted-delegation.test.ts`](../../packages/runtime/src/__tests__/execute-granted-delegation.test.ts) ‚Üí `SEVERING (reintroduces #357)`): the same paid hire as the happy-path transcript test but with `signingKeys` **omitted** ‚Äî the dormancy condition (`_signingKeys` null ‚áí producer skips minting). It asserts the hire still settles while the transcript is absent from both the result seam and the buffer. Honest classification: this proves the _runtime_ behaves correctly in both key states ‚Äî it is a negative-control fixture, **not** activation conformance, because it constructs `MotebitRuntime` directly, one level _below_ the composition root where `#357` actually lived. If the production builder dropped the keys again, this test would stay green.
- **Inc 2 ‚Äî the composition-root activation conformance** ([`money-runtime-activation.test.ts`](../../packages/molecule-runner/src/__tests__/money-runtime-activation.test.ts)): drives the **real production builder** (`defaultCreateMoneyRuntime`) with a real keypair, the real self-grant path (`selfIssueGrant` + `mintTick`), and `runMolecule`'s real `policyOverrides` composition, through a real ranked paid hire. The positive assertion binds the minted transcript to the builder's wiring ‚Äî `delegator_public_key` must equal the identity hex the builder received, and the signature must verify. The hostile case ran the defect-shaped mutation: severing the builder's `signingKeys` line (the exact `#357` diff) turns the positive test red at `toHaveLength(1)` while the hire still runs to broadcast. The positive test is the permanent guard; the severing run is the one-time proof it discriminates.

Two lessons the increments taught, kept on purpose. First, the mutation must be defect-shaped: Inc 1's perturbation (add keys to the negative fixture) only proved the fixture non-vacuous; Inc 2's (sever the production wiring, watch the _positive_ guarantee fail) reproduces the actual incident class. Second, activation tests must be **production-faithful in composition, not just in code path**: Inc 2's first draft passed `policyOverrides: {}` ‚Äî a config production never runs (`runMolecule` always feeds `DEFAULT_POLICY_OVERRIDES`) ‚Äî and failed `missing_scope` before selection, silently testing a different composition. The transcript-optional posture the tests encode is the decided **graded-evidence** model, not an accident: settlement proceeds without a transcript ([`routing-decision-transcript.md`](routing-decision-transcript.md) ‚Äî reveals, never authorizes) while the fleet-level conformance probe WARNs on its absence ‚Äî which is precisely how `#357` was detected. That is the shape Inc 3+ extends ‚Äî one behavioral-severing pair per pipeline link, each paired with the seam-reduction of principle 1 wherever the dormancy can be made structurally impossible instead.

## Provisional until hostile evidence

The class is **not** closed by this document. Doctrine can name the obligation; only hostile production-path evidence can establish that it has been met. The bar is concrete: the activation-conformance pass must catch **reintroduced** versions of `#357`, `#358`, and `#359` against the real deployable ‚Äî the dormant producer, the lost egress, the shadowed constant ‚Äî each deliberately reinjected, each turning the pass red. Status against that bar, kept precise:

- **`#357` ‚Äî met at the composition root** (Inc 2: the severed builder turns the positive test red). Sibling audit alongside it: the producer's caller surface is exactly two compositions ‚Äî the molecule-runner spend handle (guarded) and the CLI `delegate.ts` subcommand (wires keys from identity); the CLI `runtime-factory` runtime carries no `signingKeys` but is **latent, not live** ‚Äî it has no granted-spend caller, so the trigger to wire it is a granted-spend surface appearing on the daemon.
- **`#358` ‚Äî met at the consumed seam** (Inc 3a: severing `mintedTranscript = transcript` while leaving the buffer push intact ‚Äî the _exact_ incident shape, the one the pre-fix regression test stayed green through ‚Äî turns the post-`#358` result-seam assertion red at `expected undefined to be 'alice-worker'`). The guard existed since the fix; what Inc 3a added is the severing run proving it discriminates.
- **`#359` ‚Äî met at the booted entry** (Inc 3b: [`booted-entry-activation.test.ts`](../../services/relay/src/__tests__/booted-entry-activation.test.ts) spawns the **real deployed entry** ‚Äî `server.ts` as a child process with a production-shaped strict env, real env resolution, real HTTP listener ‚Äî and runs `probeSecurityBoundaries` against it from outside; every applicable boundary must report strict, and the `#359` boundary specifically must be probed, not skipped). The severing run reintroduced `#359`'s shape at today's seam ‚Äî the entry shadowing the built config (`requireDiscoverSignature = false` between builder and `createSyncRelay`) ‚Äî and both tests went red at `POST /federation/v1/discover ‚Üí 200 (strict expects 403)`. The decisive detail: under that exact severing, the static needle gate (`check-security-default-wiring`) **stayed green** ‚Äî the entry still references the canonical constant through the builder; the shadow is a post-build mutation the needle layer structurally cannot see. The booted probe is the only layer that catches it. Inc 4 added the second rung: the same test now also boots the **compiled artifact** ‚Äî `node dist/server.js`, the exact command `run.sh`, `package.json#start`, and the DEPLOY.md systemd unit all exec (freshness guaranteed by turbo's `test` ‚Üí `build` dependency). Its severing run applied the same shadow to `dist/server.js` **only**, source untouched ‚Äî an artifact-not-source defect ‚Äî and the differential came out exactly as the ladder predicts: the source rung stayed green, the compiled rung went red at `POST /federation/v1/discover ‚Üí 200 (strict expects 403)`. A defect that exists only in the built output (stale or corrupted emit, build-pipeline drift) is structurally invisible to every source-level layer; the artifact rung is the only one that observes it. Both rungs are kept deliberately: a source-green/artifact-red differential localizes the fault to the build step. The rung above completes the tier: the shipped **container image**. `publish-images.yml` now boots the built image before anything is pushed ‚Äî the `pnpm deploy`-flattened filesystem, the pinned base-image node, `run.sh` itself, with the image's own production env defaults and no overrides ‚Äî and runs [`probe-image.mjs`](../../services/relay/scripts/probe-image.mjs) from a sibling container sharing its network namespace. The probe imports the image's OWN compiled `probeSecurityBoundaries`, so the probe bytes are the artifact's bytes; a lost boundary blocks the push, meaning a bad image is never published. Its severing run derived a severed image from the real one (the `#359` shadow patched into the image's compiled `server.js`, nothing else changed) ‚Äî the pristine image probed green, the severed image probed red at `POST /federation/v1/discover ‚Üí 200 (strict expects 403)`.

### The pipeline-link ladder

With the booted-artifact tier standing, the fan-out is one behavioral-severing PAIR per pipeline link (identity‚Üíauthz‚Üípolicy‚Üíaction‚Üíreceipt‚Üísettlement‚Üítrust), each paired with seam-reduction where affordable. Status, kept precise:

- **identity‚Üíauthz ‚Äî paired.** Seam-reduction: `mintAudienceToken` (@motebit/crypto) + `check-token-mint-canonical` ended the 23-site restatement of `iat`/`exp`/`jti`/TTL. Behavioral pair: [`booted-authz-activation.test.ts`](../../services/relay/src/__tests__/booted-authz-activation.test.ts) boots the compiled artifact, provisions a device over the real HTTP API, and drives the exact client-shaped token at the real route ‚Äî accept-half (a `market:query` device token reaches market discovery: the f9d875b1 fix observed from outside; the incident hid because server tests used master while client tests mocked fetch ‚Äî this suite makes the two contract halves meet in the deployed artifact) and reject-half (the same key's `sync` token refused; unauthenticated refused). Severing runs proved BOTH directions: removing the catch-all exclusion (the exact f9d875b1 shape ‚Äî registered-but-rejected) turns the accept-half red at 401; removing the `dualAuth` registration turns the reject-halves red at 200.
- **policy‚Üíaction ‚Äî paired at the composition root** (in-process; the booted rung remains). Seam-reduction: the R4-money authority AND is single-sourced in `PolicyGate.validate` (step 8b/8c) with one audited producer of `verifiedGrant` (`verifyGrantForTurn`), statically pinned by [`check-money-authority`](../../scripts/check-money-authority.ts) ([`memory-never-confers-authority`](memory-never-confers-authority.md)). Behavioral pair: [`money-runtime-activation.test.ts`](../../packages/molecule-runner/src/__tests__/money-runtime-activation.test.ts) gains the composition-root NEGATIVES ‚Äî a revoked self-grant, an out-of-scope grant, and a ceiling-tampered grant, each driven through the REAL `defaultCreateMoneyRuntime` (not the direct-`MotebitRuntime` construction the runtime-layer negatives use), each asserting fail-closed (`requires_verified_grant` / `missing_scope`) with no selection and no rail broadcast. The gap they close: every fail-closed proof lived one layer below the builder, so a drift in the builder's policy wiring (e.g. `denyAbove` or the self-grant scope) would leave them all green.
- **Discovered by the severing, not by design:** the revocation list reaches `verifyGrantForTurn` only through the caller-threaded `params.delegation.revocations ?? []` in `executeGrantedDelegation` ‚Äî a fail-open default (`?? []`) that NO static needle covers. `check-money-authority` pins the verify call, the null-check, the scope fence, and the meter-wrap, but not revocation threading; severing `?? []` ‚Üí `[]` turns the revoked-grant negative red while the gate stays green. This is the same needle-vs-behavioral distinction #359 drew at the entry, now inside the money path: the behavioral negative is the only layer that catches a dropped revocation thread. (Deferred-with-trigger: extend the gate's needle #4 to require the revocation argument be caller-threaded, when the money path next changes.)
- **action‚Üíreceipt ‚Äî booted rung shipped** (the delegator-side egress slice stays in-process). The guarantee: an action settles ONLY on a signed `ExecutionReceipt` that verifies under Ed25519. `#357` (transcript producer dormant when the composition root didn't wire signing keys) and `#358` (minted receipt never threaded onto the result) prove the DELEGATOR-side egress in-process; those cover the producer, not the relay's settlement-authorizing gate. The relay-side gate ‚Äî `POST /agent/:id/task/:taskId/result` ‚Üí `handleReceiptIngestion`'s `verifyExecutionReceipt` ‚Üí `if (!receiptValid) return { verified: false }` (403) ‚Äî was only ever exercised in-process (`money-runtime-activation.test.ts`'s header recorded the missing booted rung). Behavioral pair: [`booted-receipt-activation.test.ts`](../../services/relay/src/__tests__/booted-receipt-activation.test.ts) boots the compiled artifact, provisions a worker over the real HTTP API, queues a free self-delegation task, and POSTs receipts at the real `/result` route ‚Äî accept-half (a valid worker-signed receipt bound to its task ‚Üí 200) and forgery-half (a byte-tampered signature ‚Üí 403). Transport auth uses the operator master token so the only variable under test is the receipt body. Severing run: `let receiptValid = await verifyExecutionReceipt(...)` ‚Üí `let receiptValid = true` + rebuild dist ‚Üí the forgery-half flips to 200 (`expected 200 to be 403`) while the accept-half holds. The gate goes red exactly when the artifact stops verifying the signature.
- **Discovered by the action‚Üíreceipt severing, not by design:** `handleReceiptIngestion` looks up the verifying key from `receipt.motebit_id` (a receipt-body field) and, on a first-pass miss against the registered key, falls back to the key EMBEDDED in the receipt ‚Äî heal-writing it onto `agent_registry.public_key` for `receipt.motebit_id`. An agent that self-delegates can mint a `task:result` token for its own task, then POST a receipt claiming a VICTIM's `motebit_id` with its own key embedded; the fallback verifies against the embedded key and OVERWRITES the victim's registered key ‚Äî a cross-identity registry-key hijack no static needle covers (every wiring string is present; the missing guard is a runtime condition). The naive fix ‚Äî bind `receipt.motebit_id === entry.task.motebit_id` ‚Äî is WRONG and was reverted before merge: routed delegation legitimately separates the task TARGET (`task.motebit_id`) from the EXECUTOR (`receipt.motebit_id`), so binding them reds ~19 delegation/credential tests (A‚ÜíB where B executes A's routed task; the internal WS/MCP ingestion sites carry the same target‚âÝexecutor shape). The correct invariant is subtler and touches the key-rotation path ‚Äî the heal-write must be bound to key material already associated with `receipt.motebit_id` (a registered DEVICE of that identity), never to an arbitrary self-signed key, or the overwrite-an-existing-key branch must be removed (heal only when no key is on file). Deferred-with-trigger to the embedded-key-fallback seam ([`verify-family-fail-closed`](verify-family-fail-closed.md)): this is a money/security path, so the arc records the gap rather than ship a hasty rebind. Trigger: the next change to `handleReceiptIngestion`'s key resolution, or a second consumer of the embedded-key heal.
- The remaining links (receipt‚Üísettlement, settlement‚Üítrust) are unpaired at the booted tier.

**Discovered by the second severing, not by design:** the market-candidates carve-out is TWO sites that must agree ‚Äî the catch-all's exclusion list and the route's own `dualAuth` registration ‚Äî and losing only the registration leaves the route **fail-open** (fully unauthenticated), a strictly worse posture than the f9d875b1 incident's fail-closed-wrong 401. No gate covers this desync class ([`check-agent-route-auth`](../../scripts/check-admin-route-auth.ts) scopes to `/api/v1/admin/*` only), and the exclusion list mixes genuinely auth-less-by-design routes (signature-is-auth, public protocol artifacts) with routes that carry their own auth elsewhere. Deferred-with-trigger: a gate (or a single-registration seam where the exclusion entry carries its expected audience and generates both halves) lands when the next carve-out is added or the next route-auth incident occurs ‚Äî whichever is first. Until then the booted authz pair behaviorally guards the one instance severed here.

The remaining honest gap: `#357`/`#358` proofs invoke their compositions in-process, and the image probe is amd64-only (the multi-arch manifest's arm64 half shares layers via the buildx cache but is never itself booted) and fires on main pushes and release tags ‚Äî a publish gate, not a PR gate. All three named incidents have now turned a pass red when deliberately reintroduced ‚Äî the bar this section set, `#359` at every tier from source to shipped image ‚Äî but the class as a whole stays open: these are three incidents on one pipeline, not the pipeline. "We have closed the composition gap" remains an aspiration, not a fact, and this doc says so on purpose.

## Scope

Governs how the repository proves itself, alongside [`agentic-era-engineering`](agentic-era-engineering.md) (coherence is the scarce resource) and [`gate-repair-instructions`](gate-repair-instructions.md) (a gate teaches its rule on contact). It is not a wire-protocol change and introduces no new type. It is the invariant that sits _above_ the gate suite: each gate proves a part; this obligation demands proof that composition-in-deployment preserves the guarantee the parts were built to make.
