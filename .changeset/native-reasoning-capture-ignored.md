Native extended-thinking capture — providers now feed the same interior-reasoning disclosure whether a model reasons via `<thinking>` tags or natively. Both `AnthropicProvider` (streaming `thinking_delta` + non-streaming `thinking` content blocks) and `OpenAIProvider` (streaming `reasoning_content`/`reasoning` deltas + non-streaming message fields) capture the native reasoning stream off the wire and merge it with the `<thinking>`-tag convention via the new `mergeReasoning` helper into `AIResponse.reasoning`. This also closes an Inc-1 gap: the Anthropic provider (the primary cloud provider) previously captured NO reasoning at all — neither native nor tags. Interior-only throughout: native reasoning is accumulated raw off the wire, never into the visible `text`, so it stays out of chat and flows only to the owner-facing `mind` register. Fail-closed — `mergeReasoning` returns null when neither source produced anything, so no reasoning → no field → no disclosure. Enabling Anthropic extended-thinking in requests (a cost/behavior decision) is deliberately NOT included; this is the capture side, immediately live for models that emit native reasoning.
