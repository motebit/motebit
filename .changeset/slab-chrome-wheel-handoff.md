**Second slice of PR 1 of the agent-surface pivot — the `/wheel` mode-flip mechanism that makes the new `motebit × virtual_browser` register actually reachable.** Without the flip mechanism, the dispatcher's polarity correction is visible but inert: users land in motebit-state-by-default (the new register renders), but there's no path to drive the page themselves. This slice ships the path. Doctrine: [`chrome-as-state-render.md`](https://github.com/hakimlabs/motebit/blob/main/docs/doctrine/chrome-as-state-render.md) § "Take-the-wheel affordance in PR 1."

**The mechanism, one event, multiple surfaces.** `/wheel` slash command + tap on the URL chip in the narration strip both dispatch the same `motebit:cobrowse-wheel` CustomEvent. The web-app handler does `machine.reclaimControl()` followed by URL-bar focus + caret-at-end. Single mode-flip mechanism, two surface affordances; future polish (explicit Take-the-Wheel button, gestures, voice) compose against the same event. Surface-determinism: dispatching the typed CustomEvent the runtime already listens to means both affordances route through the same typed-capability path; `check-affordance-routing` approves by construction.

**Closing the half-affordance gotcha.** A mode-flip without surfacing the affordance the flip unlocks is half-broken — users land in cobrowse mode and find nothing editable. The handler fires both `reclaimControl()` AND focus-the-URL-input in one gesture, exploiting the synchronous subscriber path (machine commit → chrome subscriber → DOM rebuild) so the new input is already in the DOM by the time the focus call lands. `setSelectionRange(len, len)` puts the caret at the end so the user can type to append or backspace to edit rather than erase-by-typing. The existing focus listener composes the home overlay register naturally — `/wheel` reads as "I'm going somewhere," which is exactly the home overlay's grammar.

**Chip as the spatial-natural handoff target.** The URL chip in the narration strip represents "the page motebit is on" — grabbing it means "I'll drive that page now." Promoted from a `<span>` to a `<button type="button">` so click semantics, focus-ring, and keyboard activation come from the platform; an `aria-label` of "Take the wheel — switch into cobrowse mode on {host}" makes the semantic intent legible to assistive tech. Calm-default visual treatment (borderless tinted text, transparent background, hover-lifts-opacity) inherits the chrome's history-button family so the chip reads as part of the slab-native surface, not a web-form button.

**What this slice deliberately defers.** `/back` (user → motebit) is not shipped here. Wire-wise, there's no existing single-transition path for "user yields to motebit" — the existing transitions (`reclaim_control`, `release_control`) assume the user-as-default-driver polarity. Adding a `yieldControl(by: "user")` transition is the right architectural move, but it composes with the cobrowse-as-mode reshape's "motebit waiting" indicator (which is the user-state-register's design polish, and where `/back`'s chip-tap-from-user-register lives). Ships in the next slice. Other-embodiment chrome columns (`mind`, `tool_result`, `shared_gaze`, `desktop_drive`, `peer_viewport`) stay deferred to PR N.
