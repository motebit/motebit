**First slice of PR 1 of the agent-surface pivot — the chrome's render becomes `f(controlState × embodimentMode)` and the `motebit × virtual_browser` register consumes task-step narration as its primary content.** Closes the polarity error where today's slab chrome treats the `user × virtual_browser` register as the universal default ("URL bar primary, cobrowser-shaped"); the pivot inverts the default to `motebit × virtual_browser` with task-step narration ("Reading the page" / "Filling in the form"). Cobrowse becomes a mode the user enters explicitly in a follow-up commit. Doctrine: [`chrome-as-state-render.md`](https://github.com/hakimlabs/motebit/blob/main/docs/doctrine/chrome-as-state-render.md).

**The wire.** A new `task_step_narration` chunk variant lands on both `AgenticChunk` (ai-core) and `StreamChunk` (runtime). Emitted from the loop after each iteration's `AIResponse` is captured — the proposed narration passes through `validateTaskStepNarration` against the per-turn `toolResultsLog` first, so a model claim contradicting wire-level typed truth ("Reading apple.com" while the URL is google.com) arrives at the chrome already corrected. The chunk carries `text` (validated narration) + `valid` (false when the runtime overrode a model claim — consumers may render differently for trust calibration but the contract is identical). Absent narration produces no chunk; the chrome recedes to the empty register rather than render blank text.

**The triple shape compounds.** This is the in-flight narration plumbing the typed-truth-perception triple structurally promised — wire field on `AIResponse.task_step_narration` (already shipped), `<narration>` tag in `PERCEPTION_DOCTRINE` (already shipped), producer in `extractNarrationTag` (already shipped, the four-part fix from `cf1227d0`), validator in `validateTaskStepNarration` (already shipped). The chunk closes the runtime→surface gap that was the load-bearing missing step: previously the validated narration lived only on `AIResponse` and died in the loop. The `check-typed-truth-perception` gate already verifies the triple is structurally intact (13 fields registered, all paired); this commit makes the operational pipeline match the registry-side promise.

**Streaming pass-through.** The runtime's `streaming.ts` flows the new chunk through its existing `else yield chunk;` branch — the chunk's shape is identical across the two unions so the typecheck holds. No new handling logic is required at the runtime layer; the chunk is a typed signal that downstream consumers read directly.
