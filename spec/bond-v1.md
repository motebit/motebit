# motebit/bond@1.0

An agent's **commitment bond** — its own sovereign capital, posted as a
self-signed proof-of-funds and RPC-verified by the relay, never custodied. This
spec defines the `BondCommitment` wire artifact and the law that makes it an
honest anti-sybil signal: the backing address MUST be the agent's own sovereign
identity address.

**Phase 1 is an anti-sybil staked _signal_, NOT collateral / escrow / recourse.**
A bonded agent has demonstrably tied up real, ticket-sized capital at an address
it controls — it is not a cheap throwaway. That, and only that, is what the bond
asserts. A wronged counterparty gets nothing back from a bond in phase 1 (the
recourse half is deferred — §8). Honest naming is interop law: an implementation
MUST NOT present a `BondCommitment` as secured funds or guaranteed recourse (§7).

Doctrine: [`docs/doctrine/commitment-bond.md`](../docs/doctrine/commitment-bond.md),
[`docs/doctrine/identity-binding-verification.md`](../docs/doctrine/identity-binding-verification.md),
[`docs/doctrine/off-ramp-as-user-action.md`](../docs/doctrine/off-ramp-as-user-action.md).

## 1. Scope

### Threat model & non-goals (phase 1, explicit)

**Protects against:** sybil-floor evasion. The bond is proof that the agent has
committed real, onchain, ticket-sized capital at its **own sovereign identity
address** (§2), so the identity cannot be a disposable throwaway and the capital
cannot be reused across identities. This warrants a modest relaxation of a
relying party's cold-start bar — tiered _with_ sovereign-binding, never below
(consumers decide the bar; this spec defines the artifact).

**Does NOT** (these wait for the deferred recourse half — §8): provide recourse;
deter a capitalized single-shot fraudster; protect against drain-and-default,
exit-scam, or a closed high-value pair going dark. The bond is information about
seriousness, never a guarantee.

### Custody

The relay reads the backing balance via chain RPC and **never holds, transmits,
or seizes** the capital. The bond is read, never custodied — the relay's
user-funds transmitter surface stays structurally zero. The agent may drain the
backing address at any time; doing so drops the bond's eligibility effect at the
next verification.

## 2. The anti-sybil address binding (foundation law)

A `BondCommitment` is valid **only if** its `bonded_address` equals the agent's
own sovereign address derived from `bonded_public_key`:

```
bonded_address == base58btc(bonded_public_key)        // the Solana address derivation
```

This is the whole justification. Because each sovereign identity's address is a
distinct, deterministic function of its identity key, **one wallet cannot back
many identities** — every identity must independently hold the capital at its
own address. A verifier MUST reject any commitment that breaks this binding,
fail-closed. The reference verifier (`@motebit/crypto`'s `verifyBondCommitment`)
enforces it inline; the drift gate `check-bond-address-binding` locks that the
enforcement cannot be silently removed.

`bonded_public_key` is the 32-byte Ed25519 identity key, hex-encoded. A
non-32-byte key, or a malformed hex string, is rejected.

## 3. Wire type

### 3.1 — BondCommitment

#### Wire format (foundation law)

A self-signed proof-of-funds. Field names, types, and the canonical-JSON signing
order (JCS) are binding. The bond is **self-anchoring**: it is signed by
`bonded_public_key`, and `bonded_address` is derived from that same key (§2), so
the signature proves control of the address the bond names.

```
BondCommitment {
  bond_id:            string   // UUID v7, generated by the bonding agent
  motebit_id:         string   // MotebitId of the bonding agent
  bonded_public_key:  string   // hex of the 32-byte Ed25519 identity key (the signer)
  bonded_address:     string   // base58 sovereign address == base58btc(bonded_public_key)
  bond_amount_micro:  number   // committed amount, integer USDC micro-units (1 USD = 1e6)
  asset:              string   // SettlementAsset — "USDC" at land
  chain:              string   // CAIP-2 chain id (e.g. solana:5eykt4UsFv8P8NJdTREpY1vzqKqZKvdp)
  issued_at:          number   // unix ms
  expires_at:         number   // unix ms — bond must be re-issued after this
  suite:              string   // "motebit-jcs-ed25519-b64-v1"
  signature:          string   // Ed25519 over canonical JSON of all fields except signature
}
```

## 4. Suite & signing

The single suite at land is `motebit-jcs-ed25519-b64-v1` — JCS canonicalization,
Ed25519 primitive, base64url signature. Missing or unknown values are rejected
fail-closed. The bond is signed by the **bonded private key** (the agent's own
identity key); no external signer.

## 5. Verification

A verifier MUST, fail-closed:

1. Reject an unknown `suite`.
2. Decode `bonded_public_key`; reject a malformed or non-32-byte key.
3. **Enforce the §2 binding:** reject unless `bonded_address == base58btc(bonded_public_key)`.
4. Reject unless the `signature` verifies against `bonded_public_key` over the
   canonical JSON of all fields except `signature`.

This verifies the artifact **standalone**. It does NOT bind the bond to a claimed
identity beyond the key it embeds: that `bonded_public_key` is the registered key
for `motebit_id` is the relying party's **separate** responsibility (the
`verifySovereignBinding` shape — a relay checks the registry; an offline verifier
pins the key it already trusts). And it does NOT prove the address is solvent
**now** — backing is a live RPC read (§6).

## 6. Backing freshness (staleness is an adversarial window)

A "backed" read is eventually-consistent and can be stale — an agent can drain
the address between a verifier's poll and a relying party's decision. Therefore:

- A relying party MUST bound the maximum age of a backing read it will act on.
  The reference bound is **60 seconds**; a read older than the bound is not
  "backed" for decision purposes.
- A surface that displays backing status MUST show an explicit **as-of
  timestamp** — never present a stale read as real-time.
- A relying party that grants an effect on the strength of the bond (e.g. a
  relaxed eligibility bar) MUST **re-verify backing at decision time** — a fresh
  read within the bound, else a synchronous balance check — and MUST NOT act on a
  stale "backed".

The bond also expires at `expires_at`; a verifier MAY treat an expired
commitment as unbacked regardless of balance.

## 7. Surface honesty (interop law)

Any counterparty- or operator-facing surface that shows bond status MUST present
it as a **soft anti-sybil signal**, never as secured funds or guaranteed
recourse. Framings such as "secured", "guaranteed", "escrow", "protected", "your
money back", "covered", or "recourse" MUST NOT appear as the meaning of a bond.
The honest reading is: _this agent staked a credible, withdrawable commitment — a
signal, not your money back._ (In the reference implementation a word-list gate,
`check-bond-surface-honesty`, plus a recorded human design-review checkpoint
enforce this — the word-list is necessary, the human review catches a
compliant-but-misleading layout the word-list cannot.)

## 8. Deferred — the recourse half (named-with-trigger)

The artifacts that give a bond _teeth_ — `BondCall` (a relying party calls a
bond against a defaulting agent) and `BondDefault` (a signed, public,
append-only record of a default, the sibling of `AgentRevocationFeed`) — are
**deliberately not defined in this version.** Fixing a wire shape for an
unproduced mechanism is speculative; they land when real settlement volume and
disputes exist, designed to fit that implementation. Until then a bond is a
signal, and §7 is what keeps that honest.

A hard, seizure-based guarantee would require custody (an escrow program or an
underwriter), which this spec's §1 custody rule forbids the relay from holding;
that fork is deferred behind its own trigger (a default rate that breaks the game
theory, or a materially-exposed cohort).
